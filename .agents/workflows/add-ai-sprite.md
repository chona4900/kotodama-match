---
description: 画像生成AIで作成した新キャラ画像をゲームのアニメーションとして追加する手順
---

このワークフローは、ユーザーが画像生成AIで新しいキャラクターの3コマ画像（スプライトシート）を作成した際に、それをノイズなく綺麗なアニメーションとしてゲーム内に追加するための確定手順です。

1. **画像ファイルの確認**
   ユーザーからアップロードまたは保存された新しいキャラクターの画像ファイル（例：`koryu.jpg` など横並び3コマの画像）が `kotodama-match` フォルダにあることを確認する。

2. **`index.html` の `PIXEL_ARTS` 設定を更新する**
   該当するキャラクター（例：`childB`、`childC` など）の配列を、以下の画像読み込み用のオブジェクトに書き換える。
   ※描画エンジンは既に自動透過・液晶カラー化に対応しているため、データ定義を変えるだけで動きます。

   ```javascript
   // 変更例
   childB: { type: 'image', src: 'ファイル名.jpg', frames: 3, speed: 600 },
   ```
   - `src`: 保存したファイル名
   - `frames`: 横並びのコマ数（基本は3）
   - `speed`: アニメーションの速度（600にすると少しゆったりめ。好みに合わせて調整可能）

3. **反映の確認**
   ブラウザをリロードしてもらい、指定した速度で綺麗に透過・色変換されて動くか確認する。
