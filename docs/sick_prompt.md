# Sick State AI Prompt

Use this prompt to generate the "sick/weak" animation frames for your characters. It follows the same strict rules as your previous prompts but focuses on an "unwell" appearance.

---

**Prompt:**
I have uploaded a reference image of my character. Please convert this into a 2D pixel art sprite sheet for a "sick/unwell" state in a retro handheld game.

**CRITICAL LAYOUT RULES:**
- The canvas MUST be a wide horizontal rectangle (3:1 aspect ratio).
- EXACTLY ONE ROW containing 3 frames. Crop tightly to the character.
- DO NOT draw extra art or text.

**STYLE REQUIREMENTS:**
- Resolution: 24x24 pixel grid feel per frame.
- Color Palette: Pure black and white (monochrome) only.
- Aesthetic: Sharp high-contrast pixels, GameBoy / retro digital pet style.

**ANIMATION REQUIREMENTS (Sick/Weak State):**
- **Frame 1 (Weak):** The character looks visibly exhausted. Eyes might be "half-closed" or slightly drooping. The silhouette is slightly slumped.
- **Frame 2 (Dizzy/Shaking):** The character looks dizzy. Use small "shaking" lines around the head or "swirly/spiral" eyes. The pose is even more unstable.
- **Frame 3 (Fever/Exhausted):** The character is sweating (simple drops) or has its tongue hanging out weakly. It looks heavily out of breath.

**Consistency:** Maintain the same character anatomy from the reference image, but modify the expression and posture to convey that it hasn't heard any "Kotodama" for a long time.
